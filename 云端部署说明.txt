门诊患者管理 - 云端长期部署说明

当前版本适合门诊当天使用：
- 数据保存到云端文件，服务重启后仍在
- 日期变化后自动清空
- 默认按中国时间 UTC+8 判断日期

推荐环境变量：

APP_USER=admin
APP_PASSWORD=你自己设置的密码
DATA_DIR=/var/data
AUTO_CLEAR_DAILY=true
TZ_OFFSET_MINUTES=480

Render 设置：

Build Command:
npm install

Start Command:
node server.js

需要添加 Persistent Disk：

Mount Path:
/var/data

Size:
1GB

说明：
- 当天数据会保存在 /var/data/patients-data.json。
- 第二天第一次访问或写入时，如果日期已变化，会自动清空旧数据。
- 如果你想临时内存模式，可以设置 MEMORY_ONLY=true，但服务重启后会立刻丢数据。
- 不要把 APP_PASSWORD 写进公开仓库。
